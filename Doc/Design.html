Описание методики

  1. Есть сервер DCSrvc
  2. Есть клиент

Сервер предоставляет несколько вариантов взаимодейтсвия с ним:

  • по TCP (v0, v30, v31, v32, v33)
  • по HTTP (https)

Клиент использует библиотеки. Библиотеки могут быть установлены в виде визуальных и невизуальных компонентов.

Взаимодейтвие.

Чтобы отображать в реальном времени показания датчиков сервера необходимы:

  • объект, который знает как получить данные с сервера: Connector
  • объект, который периодически опрашавает нужные датчики: DataCollector
  • объект, который хранит информацию о датчике
  • объект, который хранит список датчиков
  • объекты для визуального отображения показаний датчиков
